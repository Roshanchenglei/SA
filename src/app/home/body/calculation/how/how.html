IRAF-Based Workflow

This website primarily focuses on the analysis of astronomical data using IRAF (Image Reduction and Analysis Facility). In my workflow, I begin by downloading spectral data from the IUE MAST (Mikulski Archive for Space Telescopes). Once the data is obtained, I convert it into .fits (Flexible Image Transport System) format to make it compatible with IRAF and other astronomical analysis tools. The detailed conversion procedure is outlined below.

Format of Analyzable Data

FITS format is the most commonly used format in astronomy for multi-purpose tasks like digital storage, transmission, and data processing. This format was specifically designed for storage of human-readable ASCII (American Standard Code for Information Interchange) header, which includes information related to the observation like date of observation, the time of observation, the exposure time for that object, the basic details of the object in both the photometric and spectroscopy data files. Analysis of data in the FITS format is possible in many programming languages used in astrophysics like C, C++, Fortran, IDL (Interactive Data Language), Julia, MATLAB (Matrix Laboratory), Python, Rand many more.

Conversion to standard format

The spectral data obtained by the IUE satellite were initially processed using IUESIPS (IUE Spectral Image Processing System) software from the time of the launch of the satellite on January 26,1978. With the archived data, it became increasingly necessary for the data to be processed uniformly and calibrated uniformly to be archived as the final product of the IUE mission. Hence NEWSIPS (New Spectral Image Processing System) was developed to produce a final uniform archival data (Garhart et al., 1997; Nichols, 1997; Nichols and Linsky, 1996), for all observed objects by IUE via a standard pipeline-processing and is the second and final processing system used to calibrate IUE data resulting in an improvement in the spectral (data) calibration, signal-to-noise ratio (snr), and homogeneity for IUE data than the earlier IUESIPS processed data.

The advantages of using NEWSIPS over IUESIPS are the followings:

A great improvement is seen in the snr (signal to noise ratio) value of the spectrum while flux is conserved. Approximately 10-50% increase in the value of snr than the earlier processed spectra using IUESIPS.

Contribution due to cosmic rays is automatically removed.

The final spectrum will include an output for error estimated for each point in the flux spectrum.

NEWSIPS algorithms mainly included a background removal algorithm that determines the background level of each high-dispersion image by fitting, in succession, one-dimensional Chebyshev low/high-dispersion spectral data are available order-by-order with wavelengths uniform sampled within an order. This set of new processing algorithms in NEWSIPS has shown a significant improvement in the snr value of the processed datal, i.e. nearly up to 10-50% for most images (Nichols-Bohlin, 1990).

IUETOOLS

Once the data is downloaded it is still not ready for scientific analysis. Hence it has to be run through IUETOOLS to convert it into the standard format for analyzing astronomical data, i.e. filename. fits format through IRAF. The IUETOOLS package contains useful tools to perform data format conversion on the NEWSIPS IUE final archival data. The data files processed by the NEWSIPS pipeline- processing software is in the filename.mxlo or filename.mxhi format which basically consists of the extracted spectra that is calibrated for wavelengths, the data flags for any and noise estimates in the form of binary tables. These have to be converted to the standard filename.fits format. The procedure to integrate IUETOOLS with IRAF is followed from the STScl web- site and a brief description of the same is given below.

  1. Copy the iuetools.tar file to a suitable directory and unpack it: 
    tar xvf iuetools.tar
  2. Compile the source code by starting an IRAF session: 
            cl> ed /home/uetools/     # If files are extracted to ~/uetools
        cl> mkpkg -p tables update
  3. Build the IUETOOLS package: 
            cl> softools
        cl> ed /home/iuetools/lib/
        cl> mkhelpdb root.hd helpdb.mip

Loading and Running IUETOOLS in IRAF

  1. Start IRAF using: 
            $ xgterm &
        $ cl       # or ecl
  2. Set and load the IUETOOLS package: 
            cl> set iuetools = /home/iuetools/
        cl> cd iuetools$
        cl> task iuetools.pkg = iuetools$iuetools.cl
        cl> iuetools
  3. To load IUETOOLS automatically at IRAF startup, edit your loginuser.cl file and add: 
            set iuetools = "/home/iuetools/"
        task iuetools.pkg = "iuetools$iuetools.cl"
        reset helpdb = (envget("helpdb") // ",iuetools$lib/helpdb.mip")

About IUETOOLS Tasks

  • mxexpand: Computes the wavelength array from FITS headers and generates .fits files compatible with STSDAS and TABLES tasks.
  • mxtomultispec: Extracts spectra from FITS binary tables created by mxexpand and outputs a multispec 2D format usable in specred package.

Using IUETOOLS to Convert Files

  1. Start IRAF session again if needed: 
            $ xgterm &
        $ ecl
  2. Load IUETOOLS: 
            cl> iuetools
  3. Navigate to your data directory: 
            iuetools> cd /home/object
        iuetools> ls
        iuetools> ls *.mxlo > mxlo.dat
  4. Run mxexpand using a parameter file: 
            iuetools> epar mxexpand
        intable = @mxlo.dat     # Set this in the parameter editor

Step 4: Convert All FITS Files to 2D Format

    iuetools> !ls *.fits > fits.dat
    iuetools> epar mxtomultispec

    # In the parameter editor:
    Input  = @fits.dat
    Output = 2d//@fits.dat
    Shift  = go    (just press Enter to accept)

Step 5: Final Notes Before Scientific Analysis

This process converts the entire dataset to 2D format by prefixing the output FITS filenames with 2d. Once the tiles are converted to the 2d.fits format, they are ready for scientific analysis using IRAF.

However, before proceeding with any detailed scientific analysis, it is essential to apply certain corrections to the data files. These corrections may include flat-fielding, wavelength calibration, flux normalization, or background subtraction depending on your analysis requirements.

Step 6: Correcting for Galactic Reddening using dered

The IRAF task dered is used to correct spectral data for the effects of Galactic reddening. This step is necessary before analyzing the physical properties of astronomical sources. 

    iuetools> ls 2d*.fits > dered.dat
    iuetools> epar dered

    # In the parameter editor:
    Input   = @dered.dat
    Output  = dr//@dered.dat
    R value = E(B-V)
    A(V)/E(B-V) = 5.0
    Shift   = go    (press Enter)

Step 7: Correcting for Doppler Shift using dopcor

The IRAF task dopcor applies a redshift correction to spectral data by shifting wavelengths based on a specified redshift value. This is essential for transforming observed spectra into the rest frame of the object. 

    iuetools> ls dr2d*.fits > dopcor.dat
    iuetools> epar dopcor

    # In the parameter editor:
    Input     = @dopcor.dat
    Output    = de//@dopcor.dat
    Redshift  = [specify your redshift value, e.g., 0.031]
    # All other parameters remain unchanged.
    Shift     = go    (press Enter)
